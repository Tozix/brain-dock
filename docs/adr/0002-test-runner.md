# ADR-0002 — Тест-раннер: `bun:test` вместо Vitest

- **Статус:** Accepted
- **Дата:** 2026-06-09
- **Контекст решения:** Phase 1 (Foundation), настройка тестов

## Context

В исходном стеке ([Claude.md](../../Claude.md) §4) тест-фреймворком указан **Vitest**.
Параллельно принято решение (ADR-0001) запускать всё на **чистом Bun**. Bun имеет
встроенный тест-раннер `bun:test` с Vitest-совместимым API (`describe/it/expect`),
работающий нативно в Bun-рантайме без дополнительного слоя.

## Decision

Использовать **`bun:test`** для unit/integration-тестов. API совместим с Vitest
(`describe/it/expect`, `mock`), поэтому миграция на Vitest при необходимости тривиальна.

## Alternatives considered

- **Vitest на Bun.** Соответствует исходному стеку, богатая экосистема. Минус — второй
  раннер поверх Bun, лишний слой и зависимости при наличии нативного `bun:test`,
  потенциальные шероховатости в Bun-рантайме. Отклонено в пользу нативного раннера.
- **`bun:test` (выбрано).** Нулевая конфигурация, максимальная скорость, единый рантайм.
  Минус — отклонение от буквы ТЗ; смягчается Vitest-совместимым API.

## Consequences

- `package.json` тестовых пакетов используют `"test": "bun test"`.
- Тест-файлы импортируют `from 'bun:test'`.
- `Claude.md` §4 обновлён: Testing — `bun:test` (Vitest-compatible API).
- Если потребуется именно Vitest (специфичные плагины/среда) — заводится новый ADR,
  заменяющий этот; благодаря совместимости API правки минимальны.

> Подтверждено владельцем проекта (2026-06-09): `bun:test` остаётся.
