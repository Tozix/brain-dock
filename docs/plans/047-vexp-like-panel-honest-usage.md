## Запрос пользователя
1. Сделать интерфейс ближе к VEXP.
2. Убрать «Выбрать проект»/«Сменить проект» (проект = открытая папка) и кнопку «Собрать контекст».
3. Честность метрик: «по использованию токенов фейковые данные?» — да, «сэкономлено≈/ср. экономия 80%»
   были фиксированной оценкой-заглушкой. Убрать фейк, показывать только измеряемое.

## Сделано
**Честные метрики.** `UsageService.summary` теперь возвращает только реально измеренное —
`{ days, calls, tokensServed }` (убраны выдуманные `estTokensSaved`/`avgSavingPct`/коэффициент). Панель
показывает **Вызовов** и **Токенов отдано** — без процента «экономии».

**Реальный Edges.** `SymbolIndexService.summary` добавил `edges` (count `code_edges`); MCP
`summarize_project` печатает `Edges: N`; расширение парсит его в `IndexStatus.edges`.

**VEXP-вёрстка панели.** Статус-строка (`● brain-dock … подключено`; заголовок BRAIN-DOCK даёт нативный
title бар). Секции метрик — строками «label ↔ значение» (как VEXP): ИНДЕКС = Символы/Рёбра/Файлы +
роли; ИСПОЛЬЗОВАНИЕ = Вызовов/Токенов отдано. Блок РЕПОЗИТОРИИ: `● ⬡ alias`, путь, и счётчики
`N символов · N рёбер · N файлов`.

**Чистка кнопок.** Убраны «Выбрать проект», «Сменить проект», «Собрать контекст». «Начать индексацию»
(в состоянии «нет проекта») вместо «Проиндексировать эту папку». ДЕЙСТВИЯ: Настроить агентов,
Переиндексировать, Добавить репозиторий, Показать логи, Настройки.
