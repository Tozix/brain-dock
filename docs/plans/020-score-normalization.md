# 020 — Нормализация score в Unified Search

**Status:** Done
**Фаза:** Backlog
**Связи:** [009-unified-search](009-unified-search.md) · [003-rag-engine](003-rag-engine.md)

## Goal
Сделать ранжирование `search_everywhere` справедливым между разнородными источниками
(code/memory/knowledge/documents): сейчас они сливаются по **сырому** score, но их шкалы
различаются (code = `0.7*cosine + 0.3*keyword`, остальные = сырой cosine), из-за чего один
источник может доминировать в выдаче независимо от реальной релевантности.

## Scope
**In:**
- Min-max нормализация score **внутри каждого источника** → `[0,1]`, слияние по нормализованному
  score. Tie-break по абсолютному (`rawScore`), чтобы при равном нормализованном вперёд шёл
  более уверенный хит.
- `UnifiedResult.rawScore` (исходный score источника) — для отображения/отладки.
- Unit-тесты: нормализация в `[0,1]`, баланс источников, tie-break.

**Out:**
- Обучаемый ре-ранкер; кросс-источниковая калибровка эмбеддингов; RRF (рассмотрено — выбран min-max
  как более интерпретируемый и сохраняющий внутриисточниковый разброс).

## Этапы
- [x] Рефактор `UnifiedSearchService.search`: группировка по источнику → min-max → merge.
- [x] `UnifiedResult.rawScore`; обновить MCP/REST отображение при необходимости.
- [x] Тесты + docs (rag/Claude.md).

## Риски
- Источник с единственным слабым хитом нормализуется в 1.0 и всплывает наверх. Митигируется
  tie-break по `rawScore` (сильные источники лидируют среди равных нормализованных).

## Definition of Done
- Нормализованные score в `[0,1]`; ни один источник не доминирует только из-за шкалы.
- `bun run ci` зелёный; документация обновлена.
