# 045 — VSCode-расширение: проект = открытая папка (VEXP-style auto-project)

**Status:** Done
**Фаза:** Hosted product / client
**Дата:** 2026-06-10
**Связи:** [042-vscode-extension](042-vscode-extension.md) · [044-vscode-extension-inline-settings](044-vscode-extension-inline-settings.md)

## Проблема
Ручной шаг «Выбрать проект» + ручное добавление репозитория с путём — лишний фрикшен. Как у VEXP,
проектом должна автоматически считаться **открытая в VS Code папка**, а индексироваться — файлы из неё.

## Сделано
- `slugify(folderName)` → server-safe slug/alias (`^[a-z0-9-]+$`, fallback `workspace`), с тестами.
- `BrainDockClient.createProject(name, slug)`.
- `ensureWorkspaceProject()` в расширении: берёт `workspaceFolders[0]`, выводит slug из имени папки,
  **find-or-create** проект (по slug; при глобальном конфликте slug — суффикс), **find-or-create**
  репозиторий (`root` = путь папки, который читает локальный воркер), делает его активным и
  ставит reindex (при создании — всегда; иначе по кнопке). Прогресс-нотификация.
- **Автозапуск**: на активации, если `brainDock.autoProject` (по умолчанию `true`), есть открытая
  папка и задан ключ — расширение само адаптирует проект под открытую папку (reindex только при
  первом создании). Команда `brainDock.indexWorkspace` + кнопка «⊕ Проиндексировать эту папку»
  (главная в состоянии «нет проекта», плюс в ACTIONS). «Выбрать проект» остаётся как ручной запасной.
- Версия 0.3.0; всё локализовано (ru/en).

## Проверено
Вживую через `BrainDockClient`: `slugify` → `createProject` → `createRepository(root=папка)` →
`reindex queued`. `bun run ci` зелёный.

## Ограничение (hosted)
Воркер индексирует путь **на своей машине**. Локально (сервер на той же машине) auto-project из
открытой папки работает «как VEXP». Для удалённого сервера нужен git-/upload-коннект (P1 плана 041).

## Definition of Done
- При открытой папке и заданном ключе расширение само создаёт проект+репозиторий по имени папки и
  индексирует — без ручного «выбрать проект». `bun run ci` зелёный.
