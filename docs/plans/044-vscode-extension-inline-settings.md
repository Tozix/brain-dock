# 044 — VSCode-расширение: настройки прямо в панели

**Status:** Done
**Фаза:** Hosted product / client
**Дата:** 2026-06-10
**Связи:** [042-vscode-extension](042-vscode-extension.md) · [043-vscode-extension-polish](043-vscode-extension-polish.md)

## Проблема
Настройки (`serverUrl`/`mcpUrl`/`project`/`language` + API-ключ) приходилось искать в общих настройках
VS Code — неудобно. Нужна форма редактирования **внутри панели**. Также после смерти dev-сервера
панель показывала `fetch failed` (сервер не запущен) — это не баг расширения, но подтвердило, что
исправление `serverUrl` должно быть в один клик из панели.

## Сделано
- **Встроенная форма настроек** в webview: поля Server URL, MCP URL, Project, Language (select
  auto/en/ru) и API key (password, пусто = не менять), кнопки **Сохранить/Отмена**. Стили — через
  `--vscode-input-*`.
- Форма показывается автоматически в состоянии «не настроено» (первый запуск) и по кнопке
  **Настройки** (тумблер `settingsOpen` в провайдере, в т.ч. из состояния ошибки — чтобы сразу
  поправить `serverUrl`).
- Протокол сообщений webview→host расширен: `{type:'saveSettings'|'toggleSettings'|'cancelSettings'}`
  помимо `{command}`. Сохранение пишет в конфиг (Global) + ключ в SecretStorage.
- Кнопки «Настройки» в панели больше не открывают общие настройки VS Code, а разворачивают форму.
- Версия 0.2.0. Все строки локализованы (ru/en).

## Проверено
Рендер: «не настроено» показывает форму (serverUrl/apiKey/НАСТРОЙКИ/Сохранить), select языка
auto/English/Русский; `settingsOpen` показывает форму + Отмена + 🔑-подсказку. `bun run ci` зелёный
(151 pass / 12 skip).

## Definition of Done
- Настройки и API-ключ редактируются в панели без захода в настройки VS Code; Save применяет их и
  переподключает. `bun run ci` зелёный.
