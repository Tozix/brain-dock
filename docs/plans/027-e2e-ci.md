# 027 — e2e-CI с реальными сервисами

**Status:** Done
**Фаза:** Backlog (опционально)
**Связи:** [007-production-readiness](007-production-readiness.md) · [003-rag-engine](003-rag-engine.md)

## Goal
Прогонять интеграционные тесты против **реальных** Postgres/Qdrant/Redis (а не фейков):
ingestion→search через настоящий Qdrant и memory roundtrip через Postgres+Qdrant. Дополняет
юнит-CI, ловит интеграционные регрессии.

## Подход
Тест `*.e2e.test.ts`, **gated `RUN_E2E`** (`describe.skip` без флага → обычный `bun test` его
пропускает, реальные сервисы не дёргаются). В CI — отдельный job `e2e` с service-контейнерами:
поднимает БД, применяет миграции (`db:deploy`), ждёт Qdrant, гоняет e2e-файл с `RUN_E2E=1`.

## Scope
**In:**
- `apps/api/src/e2e/integration.e2e.test.ts`: реальный `QdrantStore` + deterministic embedder —
  ingestion→search; `MemoryService` (Postgres+Qdrant) remember→search; уборка за собой.
- Job `e2e` в `.github/workflows/ci.yml` (postgres/qdrant/redis, миграции, wait-for-qdrant).

**Out:** Ollama в CI (используем deterministic embedder); нагрузочное тестирование.

## Этапы
- [x] e2e-тест (gated `RUN_E2E`) — Qdrant ingestion→search + memory roundtrip.
- [x] Job `e2e` в workflow (сервисы + миграции + RUN_E2E).
- [x] Локальная проверка против поднятых сервисов; docs + commit/push.

## Definition of Done
- `RUN_E2E=1 bun test <e2e>` зелёный против реальных сервисов (проверено локально).
- Обычный `bun run ci` не дёргает сервисы (e2e пропущен). Workflow-job валиден.
